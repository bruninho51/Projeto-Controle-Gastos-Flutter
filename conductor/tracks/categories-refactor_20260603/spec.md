## Summary

Refatorar a feature de categorias de gastos, reduzindo o acoplamento e o tamanho do arquivo principal (`CategoriesPage`) sem fragmentar excessivamente a feature. O comportamento visual e funcional deve ser preservado integralmente.

## Context

O app Orçamentos gerencia categorias de gastos que são usadas para classificar despesas. A tela `CategoriasDeGastoPage` (já movida para `lib/features/categories/pages/`) concentra responsabilidades demais em um único arquivo: estado, integração com `ApiService`, CRUD, header customizado, dialog de criação, mapeamento de ícones, estado vazio e renderização dos cards. Isso dificulta a manutenção e evolução da feature.

Outras features do projeto (`Notifications`, `Dashboard`, `Config`) já adotam uma organização similar à proposta. Esta refatoração alinha `Categories` ao padrão estabelecido.

## Problem Description

O arquivo atual (`categorias_gastos_page.dart`) reúne em ~710 linhas:
- Gerenciamento de estado da tela
- Chamadas diretas ao `ApiService`
- Lógica de CRUD de categorias
- Header customizado (`_CategoriasHeader`, `_HeaderButton`, `_PulseDot`)
- Dialog de criação (`_showCreateCategoriaDialog`)
- Mapeamento de ícones por nome (`_getIconForCategoria`, `_containsAny`)
- Estado vazio (`_buildEmptyState`)
- Card de categoria (`_CategoriaCard`) com animações

Essa concentração aumenta o risco de regressão e dificulta futuras evoluções.
